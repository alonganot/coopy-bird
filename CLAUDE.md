# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

A browser game split across three files. No build step, no dependencies, no package manager — open `flappy_bird.html` directly in a browser.

- `flappy_bird.html` — HTML shell: canvas element, CSS, and `<script src="skills.js">` + `<script src="game.js">` (skills.js loads first)
- `game.js` — core game loop, rendering, color shop, persistence, and input handling
- `skills.js` — all skill data, state, physics hooks, and the skills shop tab. Shares game.js's globals informally (no imports) — safe because skills.js only *reads* them inside functions invoked after both scripts have run their top-level code.

## Architecture

- **Persistence** — `localStorage` via `loadData`/`saveData`. The `gameData` object holds `totalCoins`, `highScores`, `unlockedColors`/`activeColor`, `unlockedPipes`/`activePipe`, `unlockedBackgrounds`/`activeBackground`, `unlockedSkills`, `activeSkills`, and `equippedOrder`. `migrateData()` (game.js) fills in color/pipe/background/coin/score fields and delegates skill fields to `migrateSkillData()` (skills.js).
- **Game loop** — `loop()` runs via `requestAnimationFrame`. Each frame: draw background → pipes → bird → clone (if alive) → projectiles → particles → HUD → overlay → shop (if open) → `update()`.
- **State machine** — `state` is `'idle' | 'play' | 'dead'`. `shopState` is `null | 'shop'` and overlays on top of `state === 'idle'`. `shopTab` is `'colors' | 'skills' | 'pipes' | 'bg'` (tab defs in `SHOP_TABS`).
- **Shop** — `drawShop()` renders the tabbed UI (title, coin balance, `drawShopTabs()`, then dispatches to the active tab's draw function via a lookup table, falling through to the inline Colors grid); `handleShopClick()` mirrors this dispatch for clicks. `drawCloseButton()` and `renderShopMessage()` (transient buy/equip feedback, e.g. "NOT ENOUGH COINS") are each drawn once, shared across all tabs. `purchaseOrEquip(item, unlockedList, activeKey)` is the shared single-select buy/equip helper used by Colors, Pipes, and Backgrounds (Skills keeps separate multi-slot-equip logic in skills.js).
- **Color shop** — `SHOP_COLORS` array (game.js) defines all purchasable bird color schemes. `getActiveColorItem()` looks up the active color and is called inside `drawBird()`.
- **Pipes shop** — `SHOP_PIPES` array (game.js) defines purchasable pipe skins: `barrelColors`/`capColors` (gradient stop lists), `stripeColor`/`scanlineColor`/`barrelGlow`/`capGlow`/`edgeGlow` (accent colors), and `capStyle` (`'straight' | 'hazard-stripe' | 'rounded' | 'diamond-notch' | 'ringed-segments' | 'crystal-spike'`, switched on inside `drawPipe()`'s `drawCap()`). `getActivePipeItem()` looks up the active skin and is called inside `drawPipe()`. `getPipeItemRects()`/`drawPipesTab()`/`drawPipePreview()`/`handlePipesShopClick()` implement the tab (2-column grid, mirrors the skills-tab "shared rects" pattern).
- **Backgrounds shop** — `SHOP_BACKGROUNDS` array (game.js) defines purchasable scene themes: `sky` (gradient stops), `floor` (`{top, bottom}`), `horizon`/`grid` colors, `decor` (`{kind, color, count}` — kind is `'stars' | 'fireflies' | 'bubbles' | 'sandhaze' | 'snowflakes' | 'embers'`, switched on inside `drawBg()`), and optional `glowAccent` (a static radial glow, e.g. desert's sun/volcano's lava). `getActiveBackgroundItem()` looks up the active theme and is called inside `drawBg()`. The `decor` particle array (regenerated by `makeDecor()`, sized per-theme) drives all background ambience — same particle shape `{x,y,r,phase,speed}` for every theme, only the draw/motion branch differs. `getBackgroundItemRects()`/`drawBackgroundsTab()`/`drawBackgroundSwatch()`/`handleBackgroundsShopClick()` implement the tab (single-column list); equipping a theme calls `makeDecor()` to resize the particle array.
- **Skills** (skills.js) — `SHOP_SKILLS` array defines 9 purchasable skills: Dash, Shooting, Invisibility, Pocket Dimension (time slow), Shrink, Hover, Earthquake, Freeze Frame, Shadow Clone. Max 3 equipped at once via `gameData.equippedOrder` (array of skill ids, slot number = index). Equipped skills gain a charge every `chargeThreshold` score points; `activateSkill(id)` consumes a charge and starts `skillActiveTimer[id]`. Hover is hold-to-activate via `startHover()`/`stopHover()` instead of a single tap. `updateSkills()` runs once per frame from game.js's `update()` (after gravity/pipe movement) and handles timers, projectiles, earthquake pipe-sinking, and Shadow Clone physics. `attemptDeath()` replaces direct `onDeath()` calls in collision code — if a Shadow Clone is alive it swaps control to the clone instead of ending the run.
- **Input** — `handleCanvasClick(mx, my)` routes clicks to the shop or `jump()`. Keyboard: `Space` jumps; `Digit1`/`Digit2`/`Digit3` (keydown/keyup) activate the skill in that equip slot via `gameData.equippedOrder`.
